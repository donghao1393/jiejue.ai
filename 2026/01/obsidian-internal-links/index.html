Obsidian 内部链接完全指南：从基础到块引用

2026-01-10T16:14:10+04:00 | 4分钟阅读 | 更新于 2026-01-10T16:17:38+04:00

@ 董昊

Obsidian 的核心魅力在于双向链接——你的笔记不再是孤岛，而是一张相互连接的知识网络。但很多人只用到了最基础的页面链接，却不知道 Obsidian 还能精确引用到某个标题、某一行，甚至把内容直接嵌入当前页面。

基础链接：连接两个页面

最简单的内部链接就是用双方括号包裹页面名称：

[[另一个页面]]

如果想让显示的文字和实际页面名不同，用竖线分隔：

[[另一个页面|显示的文字]]

这种链接点击后会跳转到目标页面。Obsidian 会在图谱视图里把这两个页面用线连起来，你的知识网络就这样逐渐成形了。

标题引用：跳转到特定位置

当目标页面很长时，你可能希望链接直接定位到某个章节。用 # 号加标题文字就行：

[[长文档#第三章]]

点击这个链接会跳转到「长文档」页面的「第三章」标题处。如果标题有多级嵌套，可以用多个 # 号串联：

[[长文档#第三章#小节标题]]

输入 [[页面名# 后，Obsidian 会自动弹出该页面所有标题的列表供你选择，不用手动记忆。

块引用：精确到某一行

标题引用的粒度还是太粗了——如果我想引用的是一个列表项、一段话、甚至一行代码呢？这时候就需要块引用（Block Reference）。

块引用的原理是给目标内容打一个「书签」，然后在其他地方通过这个书签找到它。这个书签叫做 Block ID，格式是在目标行末尾加上 ^任意标识符。

假设你有一个叫 source.md 的文件，内容是：

- 第一项
- 这是我要引用的那一行 ^my-target
- 第三项

那个 ^my-target 就是 Block ID。现在在任何其他页面，你都可以用这个语法引用它：

[[source#^my-target]]

你不需要手动创建 Block ID。在编辑器里输入 [[source#^ 后，Obsidian 会列出该页面所有可引用的「块」。选择一个还没有 ID 的内容块后，Obsidian 会自动在源文件那一行末尾生成一个随机 ID（类似 ^a1b2c3）。

Block ID 在阅读视图（渲染后的预览模式）里是不可见的，只在编辑模式下显示，不会影响文档的整洁度。

嵌入：把内容直接显示出来

前面说的都是「链接」——点击后跳转到目标位置。但有时候你希望直接把目标内容显示在当前页面，而不是跳转。这就是嵌入（Embed）。

语法很简单，在链接前面加个感叹号 !：

![[source#^my-target]]

这会把 source.md 里带有 ^my-target 标记的那一行内容，直接渲染在当前页面。源文件修改后，这里也会同步更新。

嵌入不仅可以用于块引用，也可以用于整个页面或标题：

![[另一个页面]]           # 嵌入整个页面
![[长文档#第三章]]        # 嵌入某个标题下的全部内容
![[source#^my-target]]   # 嵌入某一行

嵌入的限制：必须独占一行

嵌入语法 ![[...]] 是块级元素，必须独占一行，无法和其他文字混排在同一行内。

比如你想写这样一句话：

根据 ![[规则文档#^rule-1]]，我们应该这样做。

这在 Obsidian 里不会生效。![[...]] 会被当作普通文本显示，而不是嵌入内容。

如果你需要行内引用，有几个替代方案：

方案一：用普通链接

根据 [[规则文档#^rule-1]]，我们应该这样做。

这会显示为可点击的链接，但不会显示实际内容。读者需要点击才能看到具体规则是什么。

方案二：使用 Dataview 插件

Dataview 插件支持行内查询，可以用 = this.file.lists.text 之类的语法在行内显示内容。但语法相对复杂，需要学习 Dataview 的查询语言。

方案三：接受独占一行的限制

调整你的写作结构，让嵌入内容自然地独占一行。比如：

我们应该遵循这条规则：

![[规则文档#^rule-1]]

基于这条规则，下一步是...

快速参考表

语法效果
[[页面]]链接到页面
[[页面|显示文字]]链接到页面，显示自定义文字
[[页面#标题]]链接到页面的某个标题
[[页面#^block-id]]链接到页面的某一行
![[页面]]嵌入整个页面
![[页面#标题]]嵌入某个标题下的内容
![[页面#^block-id]]嵌入某一行

实践建议

块引用最适合用于那些会被多处引用的内容：定义、规则、结论、关键数据。与其在多个地方重复写同样的内容，不如写一次，然后到处引用。这样当需要修改时，只改一处就够了。

但也不要过度使用。如果你发现自己在给每一行都加 Block ID，可能是笔记的组织结构出了问题——也许应该把这些内容拆分成独立的页面，用普通链接就够了。
