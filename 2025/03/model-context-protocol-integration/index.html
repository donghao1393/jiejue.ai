MCP: AI工具集成的HTTP时刻到来了？

2025-03-18T17:36:03+04:00 | 6分钟阅读 | 更新于 2025-03-18T17:43:04+04:00

@ 董昊

最近，Model Context Protocol (MCP)作为AI领域的热门话题频频出现在技术新闻中。对于被这些新闻"轰炸"却没有时间深入研究的朋友们，这篇文章将帮助你理解MCP的核心价值和实践方法，特别是解答两个常见问题：MCP是否强大到不需要微调，以及如何找到并使用开源的MCP服务。

MCP是什么？为什么突然火起来？

Model Context Protocol(模型上下文协议)是一个开放标准，它定义了AI模型与外部工具和数据源之间的通信方式。简单来说，MCP让AI模型能够更容易地读取文件、使用数据库、调用API、执行代码等操作。

为什么MCP突然受到关注？因为它解决了AI集成的一个关键痛点：标准化。

从Function Calling到MCP：AI的HTTP时刻

在MCP出现之前，大模型主要通过各自的Function Calling (FC) API与外部工具集成。这种情况与早期互联网颇为相似：

  • HTTP出现前：各网络系统使用专有协议，形成信息孤岛
  • Function Calling时代：每家AI公司都有自己的专有接口

这种类比非常贴切：MCP之于AI工具集成，就像HTTP之于互联网通信。它提供了一种统一的"语言"，让所有AI模型都能以相同的方式与外部工具交互。

MCP vs Function Calling：微调还有必要吗？

一位熟悉Function Calling的朋友问我：“难道MCP强大到不需要微调了吗？“这是个很好的问题。

Function Calling为何需要微调

微调Function Calling主要有几个原因：

  1. 提高准确性：让模型更准确地识别何时应调用特定函数
  2. 减少token消耗：微调后，模型可以在不需要完整函数定义的情况下正确调用函数
  3. 自定义响应：训练模型对函数输出做出特定格式的解释

例如，OpenAI的Function Calling示例中，微调让模型能够在看到"查询天气"这样的请求时，自动正确构造调用参数，而不需要每次都在提示中包含完整的函数定义。

MCP如何改变游戏规则

MCP通过标准化协议和更丰富的交互模式，从根本上解决了这些问题：

  1. 标准接口：MCP定义了统一的工具、资源和提示接口，使模型自然理解如何使用这些能力
  2. 客户端控制：MCP设计中的"人在环中"机制，让客户端可以审查和批准工具调用
  3. 丰富功能：除了函数调用，MCP还支持资源访问、提示模板、采样等多种交互

简而言之，MCP并不是完全消除了微调的需求，而是通过标准化大幅减少了微调的必要性。在MCP的世界里，一个设计良好的通用模型可以通过标准协议与各种工具交互，而不需要针对每种工具进行专门训练。

如何找到并使用开源的MCP服务

另一位朋友问：“怎么找到开源的MCP服务，比如天气查询？”

随着MCP生态系统迅速扩张，寻找合适的服务确实成为一个挑战。以下是几种目前可行的方法：

1. GitHub搜索

最直接的方法是在GitHub上搜索关键词。例如，如果你需要天气查询服务，可以搜索"mcp weather”。

gh search repos "mcp weather" --limit 5

这种方法能找到如adhikasp/mcp-weather、shanggqm/hefeng-mcp-weather等项目，它们分别使用AccuWeather API和和风天气API提供天气查询服务。

2. 集中式目录

社区已经建立了一些MCP服务集合：

  • Awesome列表

    • awesome-mcp-servers
    • awesome-mcp-clients

  • 专门的索引网站

    • glama.ai/mcp/servers
    • pulsemcp.com

这些目录提供了分类浏览、搜索和元数据过滤，使寻找合适的MCP服务变得更容易。

3. MCP注册中心

未来可能会出现类似于Docker Registry的MCP服务注册中心，提供认证、评级和自动发现功能。目前社区已有一些探索，如ARadRareness/mcp-registry 项目。

实践案例：安装和使用MCP天气服务

让我们通过一个具体案例，展示如何在Claude Desktop中安装和使用MCP天气服务。

选择服务

搜索GitHub后，我们发现了几个天气相关的MCP服务，这里选择adhikasp/mcp-weather ，它使用AccuWeather API提供天气查询功能。

安装步骤

  1. 获取AccuWeather API密钥

    • 访问AccuWeather API 网站
    • 注册开发者账号并创建应用
    • 获取API密钥

  2. 配置Claude for Desktop

    在配置文件中添加以下内容：

    • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
    • Windows: %APPDATA%\Claude\claude_desktop_config.json
    {
  "mcpServers": {
    "weather": {
      "command": "uvx",
      "args": [
        "--from", 
        "git+https://github.com/adhikasp/mcp-weather.git", 
        "mcp-weather"
      ],
      "env": {
        "ACCUWEATHER_API_KEY": "your_api_key_here"
      }
    }
  }
}

  3. 重启Claude Desktop

使用方法

安装后，你可以向Claude询问天气相关问题：

  • “纽约今天的天气怎么样？”
  • “接下来几小时伦敦会下雨吗？”
  • “巴黎明天的温度是多少？”

Claude会自动调用MCP天气服务查询信息并回答你的问题。

MCP生态系统的现状与未来

现状评估

MCP生态系统正在快速扩张，但仍处于早期阶段。目前有几个明显特点：

  1. 服务数量增长：各类MCP服务正在涌现，覆盖从文件系统到API集成的广泛领域
  2. 标准化进程：MCP规范在持续演进，增强互操作性
  3. 客户端支持：Claude Desktop、LibreChat等应用已支持MCP
  4. 社区活跃：开发者社区积极贡献新服务和工具

未来展望

MCP的发展路径可能会呈现以下趋势：

  1. 服务注册和发现标准化：可能出现类似"Model Service Registry"的标准服务发现机制
  2. 质量评级系统：评估服务安全性、性能和可靠性的机制
  3. 企业级采用：企业将开发私有MCP服务以安全地集成内部系统
  4. 与Function Calling的共存：FC和MCP将在不同场景下共存，并可能出现混合模式
  5. 跨模态扩展：MCP可能扩展到支持视频、音频等多模态交互

结论：AI工具集成的新范式

MCP代表了AI工具集成的新范式，它减少了集成复杂性，统一了交互方式，降低了专用微调的需求。对于开发者而言，MCP提供了创建可重用AI工具的标准方式；对于用户而言，它让AI模型变得更加强大和实用。

就像HTTP推动了Web的爆炸性发展，MCP很可能成为AI生态系统的催化剂，推动更广泛的工具集成和创新应用。无论你是开发者还是用户，现在都是了解和参与MCP生态系统的好时机。

行动建议

如果你被MCP吸引，这里有一些建议：

  1. 试用MCP服务：安装Claude Desktop并尝试一些开源MCP服务
  2. 开发简单服务：按照官方教程 创建自己的MCP服务
  3. 参与社区：加入MCP Discord by Glama 或关注GitHub讨论
  4. 贡献开源项目：改进现有MCP服务或开发新服务

随着AI工具生态系统的不断发展，MCP将在未来发挥越来越重要的作用。无论你是工程师、产品经理还是普通用户，了解MCP都将帮助你更好地驾驭AI技术的未来。
